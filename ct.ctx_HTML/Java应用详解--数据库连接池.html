数据库连接池

原理：



1. 当前web应用目录/META-INF/context.xml,只对当前应用起作用
2. 配置中，name=""属性只是标记，方便寻找，可以任意修改。方便区分，可以使用"jdbc/mysql"
3. 使用的数据池是tomcat自带的DBCP，数据库驱动应该放到tomcat的lib下
4. 在eclipse中，servers目录下的context.xml和本机/usr/local/tomcat/conf/context.xml下的两个文件不同步的竟然，所以修改Eclipse下的就好了
5. context是JNDI（Java 命名与目录接口 Java Naming and Directory Interface）的上下文对象，作用上有点像当前目录，调用lookup()就可以根据制定的JNDI的名字获得一个数据源对象，其中"java:/comp/env/"是必须有的，后面接context.xml设置的name就好。
6. 通过DataSource对象的getConnection()可以获得该数据库的连接对象。该方式获得的连接，必须手动关闭，释放资源，把该connection放回到连接池中，而不是真正的关闭相应的连接

实际操作过程：



1. 向Eclipse里Servers中Tomcat的context.xml添加:
1) <Resource
name="jdbc/mariadb"
auth="Container"
type="javax.sql.DataSource"
maxActive="100"
maxIdle="30"
maxWait="10000"
username="my"
password="1234"
driverClassName="org.mariadb.jdbc.Driver"
url="jdbc:mariadb://127.0.0.1:3306/zhangli"
/>
2. 引入驱动：把mariadb驱动复制到tomcat的lib文件夹下
3. 从连接池中获得连接实例：
1) Context context = new InitialContext();//初始化上下文环境
2) DataSource ds = (DataSource)context.lookup("java:/comp/env/java/mariadb");//取出数据源对象
3) Connection conn = ds.getConnection();
