Jury compromise poj-1015

problem :http://poj.org/problem?id=1015

本题较难，很经典，一定要摸透！

重点：


• 为什么状态选择的是 选择x个人，而不是前x个人?


◇ 在本题中，要求从 n 中选择出m个人，而且m是一个固定的数。在以往的背包等问题中，选择的物品数量没有固定值，只要求某些量达到最优解。所以，背包问题的状态的 ‘前n个’是为了保证一个物品只能够取一次，强制规定的一个顺序。而在本题中，如果没有'选择了m个'的状态的话，需要面临一下几个可能的问题：
▪ 怎么确定人数不超过m，不少于m，刚刚好是m呢？
▪ 怎么确定 一个人，是否能够对当前m人的情况有更优解？
▪ 怎么确定上面的那个人要替换谁呢？
▪ 最后又怎么获得所有 被选取的人呢？
◇ 上面一系列的问题，其实都是因为 ‘人数’ 的限制所导致的，所以我们把它加入状态中，是理所当然的！
◇ 可以想象：在没有满m人的时候，判断是很简单的，只需加上 当前人，看看解的情况就好了。但是在满m之后，就会变得很复杂。那么我们直接把问题控制到：满m之前？
◇ 这个题的子问题就是：从n个人中选择 m-1 个人...... 然后，问题的解就是: d[i][j] = max{d[i-1][j-V[i]] | 存在i,子问题也有解（前边的式子>=0 本题）}

• 边界情况的处理，很机智


• path[][]二维数组，标记结果很好用

#define ffor(i, a, b) for(int i=(a); i<(b); i++)
const int maxn=200+3;

int n, m, V[maxn], S[maxn], d[23][810];
int path[23][810];

int main(){
int kase=1;
while(scanf("%d%d", &n, &m) && (n+m)){

memset(path, 0, sizeof path);
int Mm=20*m;
d[0][Mm] = 0;

ffor(i, 1, n+1){
int a, b; scanf("%d%d", &a, &b);
V[i] = a-b;
S[i] = a+b;
}
ffor(j, 0, m){
ffor(k, 0, 2*Mm+1){
if(d[j][k] >= 0){//!!!
for(int i=1; i<=n; i++){
if(S[i] + d[j][k] > d[j+1][k+V[i]]){
int t1 = j, t2=k;
while(t1 >=1 && path[t1][t2] != i){
t2-= V[path[t1][t2]];
t1 -= 1;
}
if( t1 == 0 ){
d[j+1][k+V[i]] = S[i] + d[j][k];
path[j+1][k+V[i]] = i;
}
}
}
}
}
}
int i = Mm, j=0, k;
while(d[m][i+j]<0 && d[m][i-j]<0) j++;
if(d[m][i+j] > d[m][i-j]) k = i+j;
else k = i-j;

printf("Jury #%d\n", kase++);
printf("Best jury has value %d for prosecution and value %d for defence:\n", (d[m][k] + k-Mm)/2, (d[m][k]+Mm-k)/2);

int ans[23];
for(int z=m; z>=1; z--){
ans[z]=path[z][k];
k-= V[ans[z]];
}
